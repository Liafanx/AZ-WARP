## Changelog

Все заметные изменения проекта фиксируются в этом файле.

## [1.1.0] - 2026-04-06

### Added
- Добавлен новый режим `global-except`: весь трафик доменов идёт через WARP, кроме доменов из `exclude_domains.txt`.
- Добавлен выбор режима при установке:
  - `selective`
  - `global-except`
- Добавлено переключение режима из `warper.sh`.
- Добавлены CLI-команды:
  - `warper exclude-add DOMAIN`
  - `warper exclude-remove DOMAIN`
- Добавлена проверка существующего WARP-конфига в `/etc/wireguard/warp.conf`.
- Добавлена проверка несовместимости с AntiZapret `WARP_OUTBOUND=y`.

### Fixed
- Исправлена логика определения, что списки Gemini/ChatGPT уже добавлены.
- Исправлено поведение установки, при котором могло писаться, что Gemini уже добавлен, хотя по факту доменов не было.

### Changed
- Патч `kresd.conf` теперь зависит от режима работы.
- В режиме `global-except` встроенные списки Gemini/ChatGPT не используются.

## [1.0.24] - 2026-04-06

### Added
- Добавлена команда `warper status` для краткого статуса системы.
- Добавлены CLI-команды без меню:
  - `warper add DOMAIN`
  - `warper remove DOMAIN`
  - `warper enable gemini|chatgpt`
  - `warper disable gemini|chatgpt`
  - `warper sync`
  - `warper patch`
  - `warper doctor`
  - `warper status`
- Добавлена безопасная работа с `config.json` через `jq`.
- Добавлена проверка конфликта fake-подсети с локальными интерфейсами, маршрутами и Docker-сетями.
- Расширен `doctor`: теперь он проверяет больше системных параметров и права на чувствительные файлы.

### Changed
- `install.sh` теперь устанавливает `jq` как зависимость.
- Извлечение WARP-данных из `config.json` переведено на `jq`, с fallback на старый способ.
- README обновлён под новые команды и диагностику.
- Улучшена обработка Docker-проверок при поиске конфликтов fake-подсети.
- Подчищены shell/runtime шероховатости в CLI-командах и диагностике.
- Улучшены проверки прав и поведение в пограничных сценариях.

## [1.0.23] - 2026-04-04

### Added
- Добавлена команда `warper doctor` для диагностики состояния установки.
- Добавлено создание резервной копии `kresd.conf` перед первым патчем.
- Добавлено восстановление `kresd.conf` из backup при удалении патча и при деинсталляции.
- Добавлена безопасная загрузка файлов при обновлении утилиты.
- Добавлена проверка валидности доменов в скачанных списках перед импортом.
- Добавлена проверка списков доменов на дубликаты и мусор до записи в рабочие файлы.
- Добавлен GitHub Actions workflow с `shellcheck`.

### Fixed
- Исправлен баг смены fake-подсети в `settings_menu`: старая подсеть теперь корректно удаляется из `include-ips.txt`.
- Улучшена логика патчинга `kresd.conf`: теперь проверяется наличие точек вставки, а при их отсутствии возвращается ошибка.
- Добавлена проверка успешного запуска `sing-box` после `start/restart`.
- Добавлена валидация конфигурации через `sing-box check -c /etc/sing-box/config.json`.

### Changed
- Исключено дублирование правил `iptables` с помощью `iptables -C`.
- Unit-файлы `systemd` теперь устанавливаются в `/etc/systemd/system`.
- Чтение `warper.conf` переведено на безопасный парсинг без `source`.
- README полностью обновлён и приведён к актуальному поведению проекта.
