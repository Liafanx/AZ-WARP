# Changelog

Все заметные изменения проекта фиксируются в этом файле.

## [1.4.5] - 2026-07-20

### Fixed
- Исправлено безопасное восстановление `kresd.conf` при выключении WARPER и удалении:
  - WARPER больше не откатывает `kresd.conf` к устаревшему `kresd.conf.warper.bak`
  - если backup отличается от текущей "чистой" версии `kresd.conf`, используется очищенный текущий файл
- `backup_kresd()` теперь не одноразовый:
  - backup автоматически обновляется при изменениях upstream `kresd.conf` после обновлений AntiZapret
  - это предотвращает поломку DNS после изменений адресов `127.0.0.x` → `127.x.x.x`
- `uninstaller.sh` переведён на безопасное восстановление `kresd.conf` по той же логике

Версия поднята для совместимости с панелью https://github.com/Kirito0098/AdminPanelAZ

## [1.4.3] - 2026-06-17

### Added
- В Python API (`warper_api`) добавлены новые модули и методы:
  - **`warper_api.updates`** — управление обновлениями WARPER:
    - `check_for_updates(force=False)` — проверка наличия новой версии (кэш 60 сек, GitHub API + fallback на raw)
    - `update(timeout=600)` — синхронное обновление с ожиданием
    - `update_async()` — запуск обновления в фоне

    - `invalidate_version_cache()` — сброс кэша версии

## [1.4.1] - 2026-06-16

### Changed

- Оптимизирован запуск WARPER при больших списках доменов:
  - убран `rebuild_master_file()` из глобального старта `warper.sh`
  - пересборка master-файла теперь выполняется только там, где это действительно нужно (`sync`, ручное редактирование, update и т.д.)
- Добавлено кэширование проверки синхронизации доменов (`domains_in_sync_cached`) для ускорения:
  - главного меню WARPER
  - `warper status`
  - `warper status json` (используется веб-панелью)
- В веб-панели добавлен кэш статуса WARPER для уменьшения количества тяжёлых CLI-вызовов и снижения нагрузки при открытом dashboard.

### Fixed

- Исправлено замедление WARPER и веб-панели при больших списках доменов (например после добавления крупных каталогов вроде `google`).
- Исправлена ошибка `jinja2.exceptions.UndefinedError: 'dict object' has no attribute 'ip_ranges'` в веб-панели:
  - шаблон страницы IP-подсетей теперь корректно обрабатывает ошибку `status.error`
  - веб-интерфейс больше не падает при временной ошибке `warper status json`
- Исправлена совместная работа TUI WARPER и веб-панели:
  - `check_and_sync_warp_keys()` больше не вызывается при каждом запуске CLI-команды
  - автосинхронизация ключей теперь выполняется только при интерактивном запуске главного меню.
- Исправлена безопасность установщика веб-панели:
  - `install-web.sh` больше не удаляет безусловно `/etc/nginx/sites-enabled/default`
  - стандартный `default`-конфиг nginx удаляется только если он выглядит как штатная заглушка, а не как пользовательский сайт.

## [1.3.7] - 2026-06-05

### Added

- **📊 Статистика трафика через WARPER** — подсчёт входящего/исходящего трафика через `singbox-tun`.
  - Новый модуль `lib/traffic.sh` — чтение счётчиков ядра Linux, почасовая агрегация, хранение истории.
  - Файл `/root/warper/traffic.json` — компактное хранилище (максимум 31 день × 24 часа, ~50-80 KB).
  - Автоматический snapshot каждые 5 минут через systemd-таймер `warper-traffic-snapshot.timer`.
  - Автоматическая финализация сессии при остановке sing-box и при выключении/перезагрузке сервера.
  - CLI-команды: `warper traffic`, `warper traffic today|week|month|all`, `warper traffic today json`.
  - Статистика трафика за сегодня в `warper status` и `warper doctor`.
  - Новая страница «Трафик» в веб-панели с автообновлением каждые 30 секунд.
  - Трафик за сегодня на dashboard веб-панели.
  - При удалении WARPER с сохранением настроек — `traffic.json` сохраняется.

- **🔒 Управление HTTPS из веб-панели** — включение/выключение SSL прямо из браузера.
  - Самоподписанный сертификат — генерация одной кнопкой.
  - Let's Encrypt — ввод домена и автоматическое получение сертификата.
  - Переключение на HTTP — убирает SSL из nginx.
  - Обновление сертификата — принудительный `certbot renew`.
  - Отображение текущего режима, домена, даты истечения сертификата.
  - CLI-команды: `warper webhttps status|enable-selfsigned|enable-letsencrypt DOMAIN|disable|renew`.

- **🔄 Автосинхронизация WARP-ключей при загрузке системы**.
  - Новая функция `auto_sync_warp_keys_on_boot()` — при загрузке проверяет `/etc/wireguard/warp.conf` и молча пересобирает конфиг если ключи изменились.
  - Актуально при переустановке AntiZapret, когда выпускаются новые WARP-ключи.
  - CLI-команда: `warper warpkeysync`.

- **Новое меню HTTPS в `menus/web-menu.sh`** — подменю управления SSL/TLS с отображением текущего режима, домена, даты истечения, issuer.
- **Healthcheck веб-панели** — проверка nginx → gunicorn → Flask одной кнопкой.
- **Настраиваемые параметры безопасности** — brute-force (попытки/окно/длительность блокировки), длительность cookie.
- **Активные сессии** — просмотр успешных входов за 24ч, кнопка «Выкинуть всех» (ротация SECRET_KEY).
- **Информация о сервисе** в настройках веб-панели — аптайм, память, версии Python/Flask/Gunicorn, порты, PID.

### Changed

- **`templates/warper-autopatch.service`** переработан:
  - При загрузке: `warpkeysync` → `patch` → `ipsync` → запуск таймера трафика → первый snapshot.
  - При остановке (ExecStop): финализация сессии трафика (`trafficfinalize`).
- Навигация веб-панели: добавлена страница «Трафик», меню «Веб-панель» вынесено отдельно.
- URL в `menus/web-menu.sh` теперь корректно отображается в зависимости от HTTPS-режима:
  - HTTP → `http://IP:PORT/`
  - Самоподписанный → `https://IP:PORT/`
  - Let's Encrypt → `https://DOMAIN:PORT/`

### Fixed

- **HTTP login loop**: cookie-флаги `SESSION_COOKIE_SECURE` / `REMEMBER_COOKIE_SECURE` теперь корректно выключаются при HTTP (ранее включались при HTTPS и никогда не сбрасывались обратно).
- **`unbound variable: https_mode`** в `menus/web-menu.sh` — переменная использовалась до объявления при `set -u`.
- **Редирект после смены HTTP↔HTTPS** — теперь используется абсолютный URL с правильной схемой и сброс сессии.

### Notes

- Новые systemd-юниты (`warper-traffic-snapshot.service`, `warper-traffic-snapshot.timer`) устанавливаются автоматически через `warper update`.

## [1.3.6] - 2026-06-02

### Added

- **🌐 Веб-панель управления WARPER** — Добавлено новое меню в Web панели, туда перенесено - 🔑Учётные данные администратора. Добавлено - Информация о сервисе, 🔌Внешний порт, 🚫 Активные блокировки IP, 📋 Журнал аутентификации
  
### Fixed

- **HTTPS-установка на серверах с OPENVPN_BACKUP_TCP=y**: установщик веб-панели теперь определяет что AntiZapret использует порт 80 для backup OpenVPN-подключений, и предлагает временно остановить OpenVPN backup для получения сертификата Let's Encrypt. После получения сертификата OpenVPN автоматически восстанавливается.
- **Renewal hooks**: автоматически создаются certbot pre/post hooks которые временно останавливают OpenVPN backup при продлении сертификата. Сертификат продлевается автоматически каждые 60 дней без вмешательства администратора.
- **Улучшенная диагностика HTTPS-установки**: теперь корректно определяются все процессы слушающие порт 80 (включая несколько nginx-воркеров), и проверяется конфиг AntiZapret напрямую — не зависит от того как процесс называется в `ps`.
- **Понятные сообщения об ошибках**: если HTTPS не получился, установщик объясняет конкретные причины (firewall, DNS, занятость порта) и даёт пошаговую инструкцию.

### Changed

- В установщике улучшен интерактивный режим: чёткие подсказки при выборе вариантов в случае конфликта с OpenVPN, скрытый ввод пароля с подтверждением, безопасная генерация случайного пароля при нажатии Enter.

### Notes

При установке на сервере где AntiZapret использует `OPENVPN_BACKUP_TCP=y` — установщик попросит подтвердить временную остановку OpenVPN backup на 80 (~1 минута). После получения сертификата OpenVPN автоматически возвращается. Для автопродления через каждые 60 дней используются certbot renewal hooks — администратору ничего делать не нужно.

## [1.3.5] - 2026-06-02

### Fixed

- **MTU не менялся в режиме Slave**: `set_mtu` пытался изменить значение в `.endpoints[0].mtu`, но в Slave-конфиге `endpoints` отсутствует (Shadowsocks — не туннельный outbound). Теперь функция определяет тип конфига и меняет MTU в правильном месте — на `endpoints` для WARP/WG или на `inbounds[type=tun]` для Slave.
- В шаблоны конфигов добавлен дефолтный `mtu: 1420` на TUN-интерфейс — предотвращает использование значения по умолчанию sing-box (9000) которое может вызывать фрагментацию при использовании Shadowsocks.

## [1.3.4] - 2026-06-02

### Fixed

- **DNS-резолвинг доменных имён в outbound/endpoint sing-box**: исправлена ошибка при которой sing-box не мог корректно резолвить домены endpoint'а WireGuard / Shadowsocks-сервера потому что использовал DNS-сервер через ещё не поднятый туннель. Теперь резолвинг идёт через локальный публичный DNS (8.8.8.8) напрямую с сервера.

  Особенно актуально для:
  - WARP с endpoint в виде домена (например `engage.cloudflareclient.com:2408` вместо IP)
  - WG-режима если в `.conf` файле указан Endpoint доменом (например `vpn.example.com:51820`)
  - Slave-режима если адрес донор-сервера задан доменным именем

  Благодарность [@TripleA150](https://github.com/TripleA150) за обнаружение проблемы и предложенный фикс.

### Changed

- Шаблоны sing-box обновлены — добавлен дополнительный DNS-сервер `local` (UDP 8.8.8.8), `default_domain_resolver` переключён на него:
  - `templates/config.json.template` (WARP)
  - `templates/config-slave-master.json.template` (Slave)
  - `templates/config-wg.json.template` (WG)
- **Умная пересборка config.json при обновлении**: `warper update` теперь сравнивает хэши шаблонов до и после обновления. Если шаблон активного режима изменился — конфиг пересобирается автоматически и sing-box перезапускается. Если шаблон не менялся — sing-box не трогается, обновление происходит быстрее и без прерывания работы.

## [1.3.3] - 2026-06-01

### Added

- **🌐 Веб-панель управления WARPER** — браузерный интерфейс на Flask + HTMX (опциональная установка).
  - Полное управление через UI: домены, IP-подсети, sing-box, режимы маршрутизации (WARP/Slave/WG), все настройки.
  - Просмотр логов sing-box в реальном времени с фильтром по уровню и автообновлением.
  - Запуск warper doctor из браузера.
  - Редактор `domains.txt` и `ip-ranges.txt` с сохранением пользовательских комментариев.
  - Загрузка WG-конфигов через браузер (drag & drop).
  - Управление WARP-ключами: выбор источника, генерация новых.
  - Проверка обновлений WARPER на dashboard с одной кнопкой "Обновить" (показывает прогресс в реальном времени через Server-Sent Events).
  - Адаптивный тёмно-зелёный дизайн, мобильный и десктоп.
  - Офлайн-зависимости (Tailwind CSS, HTMX встроены — не зависят от CDN при работе).
- **Безопасность веб-панели**:
  - bcrypt-хеши паролей в `web/data/users.json` (chmod 600, доступ только root).
  - SECRET_KEY Flask в `web/data/secret.key` (генерируется автоматически), ротируется при смене пароля → все активные сессии становятся невалидными.
  - Защита от brute-force: 10 попыток / 10 минут → блокировка IP на 15 минут (persistent на диске).
  - Timing-safe проверка пароля (всегда выполняется bcrypt с одинаковым cost).
  - CSRF-защита через проверку Origin/Referer.
  - X-Real-IP только от nginx — невозможно подделать заголовком.
  - Security-заголовки nginx: X-Frame-Options, X-Content-Type-Options, HSTS (для HTTPS).
  - Аудит-лог авторизации в `web/data/auth.log` с автоматической ротацией.
  - Cookie: HttpOnly, SameSite=Lax, Secure при HTTPS.
- **Установщик веб-панели** (`install-web.sh`):
  - Выбор порта с проверкой занятости.
  - HTTPS опционально: Let's Encrypt с доменом или самоподписанный сертификат.
  - Скрытый ввод пароля с подтверждением или автогенерация безопасного пароля.
  - Интеграция в основной `install.sh` (опциональный запрос установки веб-панели).
- **CLI-команды для управления веб-панелью**:
  - `warper webpass` — интерактивная смена логина/пароля.
  - `warper webpass USER PASS` — неинтерактивная смена.
  - `warper webpass --reset` — полный сброс, создаёт admin со случайным паролем.
  - `warper webpass --unblock` — снять блокировки IP.
  - `warper webupdate` — обновление файлов веб-панели.
- **Меню веб-панели в `warper` (пункт `W`)**:
  - Установка/удаление веб-панели прямо из главного меню.
  - Смена логина/пароля, сброс блокировок IP, смена порта.
  - Просмотр логов сервиса и лога авторизаций.
  - Управление сервисом (старт/стоп/рестарт).
- **Новые CLI-команды для автоматизации и веб-панели**:
  - `warper toggle` — включить/выключить WARPER.
  - `warper mode warp|slave|wg [args]` — смена режима маршрутизации из CLI.
  - `warper subnet X.X.X.0/M` — изменение fake-подсети.
  - `warper loglevel debug|info|warn|error` — log level sing-box.
  - `warper mtu N` — MTU sing-box.
  - `warper autopatch on|off` — управление автопатчем DNS.
  - `warper fullvpn on|off` — FullVPN WARP-резолвинг.
  - `warper iproutemode antizapret|all_vpn|all` — режим применения IP-маршрутов.
  - `warper ipexport on|off` — экспорт CIDR в AntiZapret.
  - `warper warpkey list|generate` — управление WARP-ключами.
  - `warper wgconfig list` — список доступных WG-конфигов.
  - `warper status json` — статус в JSON-формате (используется веб-панелью).
  - `warper logs [N]` — последние N строк логов sing-box.
- **Единое обновление**: команда `warper update` теперь обновляет и WARPER, и веб-панель (если установлена). Запускается из CLI, главного меню или из браузера.
- **Сохранение пользовательских комментариев**:
  - `domains.txt`: комментарии (`#`) и пустые строки в пользовательском блоке сохраняются при `warper add/remove`, переключении списков Gemini/ChatGPT, синхронизации.
  - `ip-ranges.txt`: комментарии и пустые строки сохраняются при `warper ipadd/ipremove`.

### Changed

- `uninstaller.sh` теперь автоматически удаляет веб-панель (сервис, nginx-конфиг, файлы) перед удалением WARPER.
- `rebuild_master_file()` переписан для сохранения пользовательских комментариев в `domains.txt`.
- `insert_user_domain()` вставляет домен без пересборки файла — сохраняет форматирование и комментарии.
